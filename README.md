# wsc-image

> AI 生图 & 视频提示词生成 Skill，适用于 Claude Code / Trae 等支持 Claude Skills 的 AI 编程工具。

将你口语化、模糊的描述，转化为可直接投入 AI 生图/视频工具的高质量提示词。支持参考图分析、多轮迭代优化、多模型格式适配。

---

## 适用平台

- **Claude Code**（推荐）— 完整支持 AskUserQuestion 弹窗交互
- **Trae**（推荐）— 同样支持 Skills 机制，交互体验一致
- 其他支持 Claude Skills 协议的工具

---

## 核心能力

| 能力 | 说明 |
|------|------|
| 口语转提示词 | 把「帮我生成一张双十一海报，高级感」转化为完整的结构化提示词 |
| 参考图分析 | 上传参考图，自动分析风格/构图/色调/光影，基于分析生成提示词 |
| 弹窗追问 | 信息不足时弹出选择题，单选/多选智能判断，不打断对话节奏 |
| 迭代优化 | 说「差点意思」「不够好」，最多3轮追问，每轮更精准 |
| 多模型适配 | 默认即梦/豆包/Flux 中文格式，自动识别 MJ/SD 切换对应格式 |
| 经验积累 | 每次迭代记录问题和优化点，持续提升提问精准度 |

## 运行架构

### 整体流程

```
用户输入
    │
    ├─ 有参考图？ ──→ Step 0: 分析参考图（风格/构图/色调/光影）
    │                         │
    │                         ↓
    │               输出分析摘要 + 确认保留/调整内容
    │
    ↓
Step 1: 信息收集
    │
    ├─ 信息充足？ ──→ 直接生成
    │
    └─ 信息不足？ ──→ AskUserQuestion 弹窗（4-5个问题）
                              │
                              ↓
Step 2: 生成提示词
    │
    ├─ 用户满意？ ──→ 结束
    │
    └─ 不满意？ ──→ Step 3: 迭代优化（最多3轮）
                              │
                              ├─ Round 1: 问题定位 + 主体细节 + 期望方向
                              ├─ Round 2: 核心问题 + 构图调整 + 光影色彩
                              └─ Round 3: 综合所有信息，最终版本 + 迭代总结
```

### 弹窗交互设计

弹窗是这个 Skill 的核心交互机制，通过 `AskUserQuestion` 工具实现。

**单选 vs 多选的判断逻辑：**
- 互斥选择（只能是其中一种，如画面风格、色调方向）→ `multiSelect: false`
- 可叠加组合（多个都要，如氛围关键词、需要改进的点）→ `multiSelect: true`

**初始弹窗包含 4-5 个问题：**
1. 图片类型（单选）— 海报/插画/图标/产品图
2. 画面风格（单选）— 写实/3D渲染/插画/动画卡通
3. 主色调（单选）— 温暖/冷色/黑金/高对比
4. 氛围调性（多选）— 科技感/高级感/热闹喜庆/清新
5. 构图视角（多选）— 中心构图/仰视/留白/俯视

### 参考图分析机制

当用户上传图片或说「参考这张」「类似这种风格」时触发：

1. 模型对图片进行多维度视觉分析
2. 输出结构化分析摘要（风格/构图/色调/光影/氛围）
3. 询问用户保留哪些元素、主体内容是什么
4. 将分析结果直接嵌入生成的提示词中

这个机制跳过了初始弹窗中的风格/色调问题（因为已经从参考图中获取），只补充缺失的主体信息。

### 迭代优化机制

触发词识别：「差点意思」「不够好」「感觉不对」「再优化」「太普通了」等。

每轮迭代都通过弹窗收集更精准的信息：
- **Round 1**：定位问题（多选）+ 主体细节（多选）+ 期望方向（单选）
- **Round 2**：核心问题（多选）+ 构图调整（多选）+ 光影色彩（单选）
- **Round 3**：综合前两轮，生成最终版本，不再追问

第3轮结束后，将本次迭代记录写入 `memory/iteration_log.md`，包括：
- 原始需求
- 每轮追问内容和用户回答
- 首次提示词缺失了什么
- 下次同类需求应优先追问的维度

### 多模型格式适配

| 用户说 | 输出格式 |
|--------|---------|
| 不提模型（默认） | 中文自然语言，适配即梦/豆包/Flux |
| 「SD」「SDXL」「SD1.5」 | 英文逗号标签 + Negative prompt |
| 「MJ」「Midjourney」「Niji」 | 英文关键词 + `--ar --v` 参数 |
| 「Flux」「即梦」「豆包」「Gemini」 | 中文或英文自然语言 |

### 经验库（image_patterns.md）

记录了 9 种高频图片风格的提示词结构公式和关键词：

1. 写实摄影风格
2. 插画/概念艺术
3. 3D 商业渲染
4. UI 图标/界面元素
5. 3D 商业海报/C4D 渲染
6. Pixar/迪士尼动画风格
7. CGI 科技感海报
8. 卡通图标（F1/Flux 模型）
9. 毛绒/黏土/充气风格

每次生成前优先参考经验库，匹配对应风格的结构公式。

## 安装使用

### Claude Code

将整个 `wsc-image` 文件夹放入 Claude Code 的 skills 目录：

```
~/.claude/skills/wsc-image/
```

触发方式：直接描述需求，如「帮我生成一张...」，或使用 `/wsc-image` 命令。

### Trae

将 `wsc-image` 文件夹放入 Trae 的 skills 目录（路径参考 Trae 文档），触发方式相同。

## 主体描述规范

每次生成的提示词必须包含主体的完整描述，涵盖：

| 维度 | 示例 |
|------|------|
| 外观/颜色 | 橘白色短毛、哑光黑色机身搭配玫瑰金装饰条 |
| 形态/比例 | 圆润的脸庞、流线型头戴式设计 |
| 姿态/动作 | 慵懒地趴在窗台上、悬浮于画面中央 |
| 表情/状态 | 眼神锐利、耳罩边缘散发金色环形发光 |
| 特征细节 | 琥珀色眼睛、毛发光泽、耳机金属装饰条 |
