# wsc-image
> AI 生图、视频提示词生成 & 修图/P图提示词 Skill，适用于 Claude Code / Trae 等支持 Claude Skills 的 AI 编程工具，隶属于 **WSC Creative Suite** 系列。

将你口语化、模糊的描述，转化为可直接投入 AI 生图/视频工具的高质量提示词。支持参考图分析、修图/P图场景、无限制迭代优化、针对 SDXL/SD/MJ 的深度多模型格式适配。

## 核心能力

| 能力 | 说明 |
|------|------|
| 口语转提示词 | 把「帮我生成一张双十一海报，高级感」转化为完整的结构化提示词 |
| 修图提示词 | 提供图片 + 说「修图/调色/电影感」，输出光影/氛围增强的精准指令 |
| P图提示词 | 保留原图人物，换背景/换场景/三宫格合成，自动保护五官比例 |
| 参考图分析 | 上传参考图，自动分析风格/构图/色调/光影，并弹窗确认创作意图 (A/B/C/D) |
| 弹窗追问 | 信息不足时弹出选择题，单选/多选智能判断，不打断对话节奏 |
| 迭代优化 | 说「差点意思」「不够好」，支持无限制细节追问，每轮更精准 |
| 多模型适配 | 拥有专属 SDXL/SD/MJ 提示词标准轴，自动适配主流模型语法 |
| 经验积累 | 每次迭代记录问题和优化点，确立高阶提示词生成逻辑 |

## 运行架构

### 整体流程

```
用户输入
    │
    ├─ 有图片 + 修图/P图意图？ ──→ 场景 D：修图/P图提示词
    │                                    │
    │                         ┌──────────┴──────────┐
    │                      修图模式               P图模式
    │                   （光影/氛围增强）     （换背景/合成/三宫格）
    │
    ├─ 有图片 + 参考风格？ ──→ 场景 C：参考图四维分析 → 弹窗意图确认(A/B/C/D)
    │
    ├─ 静态画面描述？ ──→ 场景 A：7层结构提示词生成
    │
    └─ 动态内容描述？ ──→ 场景 B：分镜视频提示词生成
    
所有场景 → 信息不足时 AskUserQuestion 弹窗 → 无限迭代优化
```

### 场景 D 五维保护原则（修图/P图）

```
① 保留优先  — 严格保留原图人物五官、脸部比例、肤色
② 原比例    — 不改变画面宽高比（除非用户明确要求）
③ 无缝融合  — 新增元素与保留主体在光影/色调/透视上匹配
④ 不过度处理 — 磨皮保留真实纹理，不过度平滑，不网红化
⑤ 拒绝漂移  — 不改变人物年龄、种族、骨相特征
```

### 弹窗交互设计

弹窗是这个 Skill 的核心交互机制，通过 `AskUserQuestion` 工具实现。

- 互斥选择（风格、色调、平台）→ `multiSelect: false`
- 可叠加组合（氛围词、需要改进的点）→ `multiSelect: true`

### 多模型格式适配

| 用户说 | 输出格式 |
|--------|---------|
| 不提模型（默认） | 中文自然语言，仅适配即梦 |
| 「SDXL」「Illustrious」「Pony」 | 基于双编码器优化的英文标签 + 叙事短句 |
| 「SD」「SD1.5」 | 经典英文标签堆砌 + 高权重负向提示词 |
| 「MJ」「Midjourney」「Niji」 | 英文关键词 + `--ar --v` 参数 |

## 安装使用

### Claude Code

将整个 `wsc-image` 文件夹放入 Claude Code 的 skills 目录：

```
~/.claude/skills/wsc-image/
```

触发方式：直接描述需求，如「帮我生成一张...」「帮我修图」「换一下背景」，或使用 `/wsc-image` 命令。
