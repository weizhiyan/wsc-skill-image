# wsc-image
> AI 生图 & 视频提示词生成 Skill，适用于 Claude Code / Trae 等支持 Claude Skills 的 AI 编程工具，隶属于 **WSC Creative Suite** 系列。

将你口语化、模糊的描述，转化为可直接投入 AI 生图/视频工具的高质量提示词。支持参考图分析、无限制迭代优化、针对 SDXL/SD/MJ 的深度多模型格式适配。

## 运行架构

### 整体流程

```
用户输入
    │
    ├─ 有参考图？ ──→ Step 0: 智能参考图分析（四维度：风格/色彩/光影/构图）
    │                         │
    │                         ↓
    │               弹窗确认意图 (A/B/C/D 模式)
    │
    ↓
Step 1: 信息收集
    │
    ├─ 信息充足？ ──→ 直接生成
    │
    └─ 信息不足？ ──→ AskUserQuestion 弹窗
                              │
                              ↓
Step 2: 生成提示词 (自动适配 SDXL/SD/MJ 等格式)
    │
    ├─ 用户满意？ ──→ 结束
    │
    └─ 不满意？ ──→ Step 3: 无限制迭代优化
                              │
                              └─ 每一轮都触发精准弹窗，直到用户满意
```

### 弹窗交互设计

弹窗是这个 Skill 的核心交互机制，通过 `AskUserQuestion` 工具实现。

**单选 vs 多选的判断逻辑：**
- 互斥选择（只能是其中一种，如画面风格、色调方向）→ `multiSelect: false`
- 可叠加组合（多个都要，如氛围关键词、需要改进的点）→ `multiSelect: true`

**初始弹窗包含 4-5 个问题：**
1. 图片类型（单选）— 海报/插画/图标/产品图
2. 画面风格（单选）— 写实/3D渲染/插画/动画卡通
3. 主色调（单选）— 温暖/冷色/黑金/高对比
4. 氛围调性（多选）— 科技感/高级感/热闹喜庆/清新
5. 构图视角（多选）— 中心构图/仰视/留白/俯视

### 参考图分析机制

当用户上传图片或说「参考这张」「类似这种风格」时触发：

1. 模型对图片进行多维度深度分析（视觉风格、色彩、光影、构图）
2. **v2.1 新增**：通过弹窗让用户选择 A (全保留)、B (只要质感)、C (仅灵感)、D (自定义) 模式。
3. 结合用户意图，将提取的参数完美融入新提示词。

### 迭代优化机制

触发词识别：「差点意思」「不够好」「感觉不对」「再优化」「太普通了」等。

**v2.1 更新**：删除了 3 轮迭代限制。系统现在支持**无限次迭代**，只要用户表达不满，每一轮都会通过弹窗收集更精准的信息，直到达成理想效果。

### 多模型格式适配

| 用户说 | 输出格式 |
|--------|---------|
| 不提模型（默认） | 中文自然语言，适配即梦/豆包/Flux |
| 「SDXL」「Illustrious」「Pony」 | 基于双编码器优化的英文标签 + 叙事短句 |
| 「SD」「SD1.5」 | 经典英文标签堆砌 + 高权重负向提示词 |
| 「MJ」「Midjourney」「Niji」 | 英文关键词 + `--ar --v` 参数 |

### 经验库（image_patterns.md）

记录了 9 种高频图片风格的提示词结构公式和关键词，v2.1 版本在此基础上增加了对 SDXL 材质表现的深度理解。

## 安装使用

### Claude Code

将整个 `wsc-image` 文件夹放入 Claude Code 的 skills 目录：

```
~/.claude/skills/wsc-image/
```

触发方式：直接描述需求，如「帮我生成一张...」，或使用 `/wsc-image` 命令。
